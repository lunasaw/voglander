# 1.0.9 图像资产与采集验收报告（本地证据版）

## 已验证

- 后端图像/任务/迁移专项 reactor 测试：493 tests，0 failures；另有 integration provider 测试 7 tests，0 failures；编译链路成功。
- SQLite image migration：内存库重复执行创建 `tb_image_asset`、`tb_image_asset_source`、`tb_image_collection_config`，700 段菜单 11 条，且无 DROP。
- 前端 image/task API、路由、纯函数、Blob preview 单测：14 files / 32 tests，0 failures；`pnpm check`、`pnpm lint`、`pnpm build:antd` 均成功。构建仅有第三方 `@vueuse/core` Rolldown 注解警告。
- 前端资产页已包含统计/筛选/图库/列表/上传幂等重试/详情/下载/删除/SSE 刷新；采集页已包含机位懒加载、ONCE/SCHEDULED 校验、通用任务控制、SSE 刷新和深链。
- 日志和指标约束测试覆盖图像 code/identifier 脱敏及 handler/snapshot/storage/orphan/reconciliation 指标注册。
- 任务/架构/OpenAPI/日志扫描测试 15 tests，0 failures；对图像/任务生产源码 130 个文件执行 placeholder、敏感日志、绝对路径和 URL query 静态扫描通过。

## 未完成的外部证据

- durable task engine 变更仍有 15 项待验收，故不能宣称前置变更整体完成。
- 当前环境没有在线 GB28181 通道/ZLMediaKit，无法执行真实 startLive/getSnap/stopLive POC。
- 当前环境没有 PostgreSQL 服务；MySQL 客户端存在但没有可用服务，三数据库 migration 只能保留脚本和 SQLite 证据。
- 浏览器自动化启动在本机 30 秒超时，故真实 E2E、视觉/响应式/无障碍截图、双实例重启、故障注入、试点发布尚未执行。

## 风险与后续

上线前必须补齐上述外部证据，并在真实共享存储上验证 reconciliation、补偿和多实例 lease。后续可增加 MinIO/S3 provider、retention scheduler 与 ACL policy；这些不属于 1.0.9 首期实现。
