# Voglander 开发日志 2026-06-03 ~ 06-08

## 需求背景

本周期围绕「让平台能被看见、被联调、被级联」三个方向推进，交付了三个版本：

1. **1.0.4 级联下级平台**：让本平台作为下级，主动注册到上级国标平台，并把本地设备目录、PTZ 控制、实时视频流向上转发
2. **1.0.5 GB28181 可视化后端**：为前端提供完整的直播编排、实时事件推送、告警、PTZ、回放、鉴权能力，打通「点开设备即看直播」的全链路
3. **1.0.6 GB28181 协议验证台**：内置一套可自环联调的协议测试工具，无需真实摄像头即可验证注册、目录、控制、告警等国标流程

同时对线上稳定性做了多项加固（缓存、Redis 容错、消息链路追踪）。

## 已交付功能

### 🟢 级联到上级国标平台（1.0.4）

**用户价值**：本平台可作为下级接入更上层的国标监控平台（如公安/应急指挥中心）。配置好上级信息后，平台会自动注册上去，上级能查询到本地全部设备目录、下发 PTZ 控制、调取实时视频流。

**解决了什么问题**：此前平台只能作为「服务端」接入下级摄像头，无法反向接入上级。新版补齐了级联客户端能力——自动周期注册、注册状态同步、目录与设备信息应答、PTZ 指令透传给真实摄像头、双向媒体流转发（上级点播时本地拉流并通过媒体服务器转推上去，挂断时自动清理）。

**当前限制**：级联通道与本地协议验证台为互斥模式，同一时刻只启用其一。

---

### 🟢 实时直播一键编排（1.0.5）

**用户价值**：前端点开任一设备通道即可拉起实时��播，多人同时观看同一路时自动复用同一条流（不重复占用摄像头和带宽），最后一个观看者离开后延迟自动回收，节省资源。

**解决了什么问题**：旧实现缺少统一的直播会话编排，重复点播会反复向摄像头发起 INVITE。新版引入直播流注册表与引用计数，多节点部署下通过 Redis 原子计数保证「同一路只开一次、计数归零才关」，并保证计数不会因异常降到负数。

---

### 🟢 实时事件推送（SSE）（1.0.5）

**用户价值**：设备上线/下线、直播就绪、媒体节点故障、新告警等事件可实时推送到前端页面，无需手动刷新。

**解决了什么问题**：缺少服务端主动推送通道。新版提供 SSE 事件总线，支持多节点跨实例扇出（任一节点产生的事件，连在其他节点的前端也能收到），带心跳保活与连接数上限保护，避免单机连接耗尽。

---

### 🟢 告警全链路落地（1.0.5）

**用户价值**：设备上报的告警会被完整记录、可查询、可确认（ack），并实时推送到前端。

**解决了什么问题**：此前告警事件仅记录日志、不入库、不可追溯。新版补齐告警表、告警管理与确认能力，告警产生即落库并推送。

---

### 🟢 PTZ 控制 / 录像回放 / 接口鉴权（1.0.5）

**用户价值**：
- PTZ 云台控制接口支持方向+速度的标准指令，前端用统一词表调用
- 录像回放控制（播放/暂停/拖动）接通
- 所有接口启用 JWT 鉴权（支持 Bearer 头与 query token 两种方式），并可配置跨域白名单，提升安全性

**解决了什么问题**：补齐了可视化前端依赖的控制类接口与统一的访问鉴权，此前接口无认证保护。

---

### 🟢 直播会话可靠性兜底（1.0.5）

**用户价值**：点播长时间没拉起流时不会一直卡住占用资源，会自动标记失败并释放，等待中的请求得到明确反馈。

**解决了什么问题**：INVITE 长期处于「邀请中」无人收尾会造成会话泄漏。新版定时扫描超时会话置为失败并清理等待队列；媒体节点故障时批量关闭其上的直播流并通知前端。

---

### 🟢 GB28181 协议验证台（1.0.6）

**用户价值**：开发与联调人员无需接入真实摄像头，即可在平台内模拟一个国标设备，验证注册、保活心跳、目录查询、设备信息、PTZ 控制、告警上报、配置查询等完整流程，协议消息实时可视化展示。

**解决了什么问题**：国标联调长期依赖真实硬件，门槛高、复现难。新版内置 Lab 客户端（本机自环：模拟设备 5061 ↔ 平台 5060），通过 SSE 把协议交互实时投递到前端验证台页面，并对自环路径做了专门兜底（平台给自己当目标设备时正确握手）。整套能力通过开关门控，默认仅在开发环境开启，不影响生产。

## 稳定性加固

### 🔧 缓存管理器冲突修复

设备/媒体节点等业务缓存曾因依赖库的默认缓存管理器抢占，导致「找不到名为 device 的缓存」并静默关闭缓存清理。已显式指定使用 Redis 动态缓存管理器并标记为首选，彻底消除冲突。

### 🔧 SIP 消息链路追踪

新增协议层消息追踪：在出站/入站咽喉点统一打印完整消息体与 callId/deviceId，落到独立日志文件，可按 callId/deviceId 跨节点跨线程串联一次完整的点播或控制消息路径，大幅降低国标问题定位成本（可一键关闭）。

### 🔧 Redis 容错与配置加固

修正了 Redis 超时单位、直播会话回收在 Redis 故障时的兜底逻辑，以及集成配置中的重复键与节点绑定问题（此前会阻断启动）。SQLite 支持 clone 即跑的自动建表初始化，运行期产物移出版本控制。

### 🔧 设备注册校验修正

GB28181 设备注册校验改为校验消息发送方（From）而非平台自身（To），修正了一类注册失败问题。
