# 1.0.10 菜单结构调整

## 变更概述

本次变更修复了菜单ID冲突问题，并优化了菜单结构，将级联管理功能移至设备管理模块下。

## 问题描述

在原有SQL脚本中存在严重的菜单ID冲突：

1. **级联管理**（Cascade）使用ID 600作为一级目录
   - 601: 级联平台列表（CascadePlatform）
   - 602: 级联通道映射（CascadeChannel）

2. **任务管理**（TaskManagement）也使用ID 600作为一级目录
   - 601: 任务中心（TaskCenter）
   - 60101, 60102: 任务按钮权限

由于ID冲突，后插入的任务管理菜单会覆盖级联管理菜单，导致级联功能无法在前端菜单中显示。

## 变更内容

### 1. 菜单结构调整

**调整前：**
```
级联管理 (600) - 独立一级目录
├── 级联平台列表 (601)
└── 级联通道映射 (602)

任务管理 (600) - ID冲突！
└── 任务中心 (601)
```

**调整后：**
```
设备管理 (500)
├── 设备列表 (501)
├── 设备通道列表 (502)
├── 级联平台管理 (601) ← 新增
└── 级联通道管理 (602) ← 新增

任务管理 (800) ← ID变更
└── 任务中心 (801) ← ID变更
    ├── 任务查询 (80101)
    └── 任务控制 (80102)
```

### 2. ID变更映射表

| 功能 | 旧ID | 新ID | 变更说明 |
|------|------|------|----------|
| 级联管理目录 | 600 | - | 已删除（合并到设备管理） |
| 级联平台列表 | 601 (parent: 600) | 601 (parent: 500) | parent_id变更，路径调整 |
| 级联通道映射 | 602 (parent: 600) | 602 (parent: 500) | parent_id变更，路径调整 |
| 任务管理目录 | 600 | 800 | ID变更以消除冲突 |
| 任务中心 | 601 | 801 | ID变更 |
| 任务查询按钮 | 60101 | 80101 | ID变更 |
| 任务控制按钮 | 60102 | 80102 | ID变更 |

### 3. 路径变更

| 菜单 | 旧路径 | 新路径 |
|------|--------|--------|
| 级联平台列表 | `/cascade/platform` | `/device/cascade/platform` |
| 级联通道映射 | `/cascade/channel` | `/device/cascade/channel` |
| 任务中心 | `/task/center` | `/task/center` (不变) |

## 迁移步骤

### 对于新部署

直接执行对应数据库的主SQL脚本：
- MySQL: `sql/voglander.sql`
- PostgreSQL: `sql/voglander-postgresql.sql`
- SQLite: `sql/voglander-sqlite.sql`

### 对于已有数据库

执行对应的迁移脚本：

**MySQL:**
```bash
mysql -u root -p voglander < sql/migration/mysql/1.0.10-menu-restructure.sql
```

**PostgreSQL:**
```bash
psql -U postgres -d voglander -f sql/migration/postgresql/1.0.10-menu-restructure.sql
```

**SQLite:**
```bash
sqlite3 app.db < sql/migration/sqlite/1.0.10-menu-restructure.sql
```

## 迁移脚本说明

迁移脚本按以下顺序执行：

1. **删除旧的级联管理菜单**
   - 删除 tb_role_menu 中的关联
   - 删除 tb_menu 中的旧记录（ID 600, 601, 602及其子菜单）

2. **插入新的级联管理菜单**
   - 将级联平台管理（601）和级联通道管理（602）作为设备管理（500）的子菜单插入
   - 更新路径为 `/device/cascade/*`

3. **删除旧的任务管理菜单**
   - 删除可能存在的ID冲突记录

4. **插入新的任务管理菜单**
   - 使用新ID 800系列创建任务管理菜单

5. **更新角色权限**
   - 为管理员角色分配所有新菜单的访问权限

## 影响范围

### 后端

- **无需修改**：菜单由数据库动态加载，ID变更对后端无影响
- 权限码保持不变：
  - `Cascade:Platform:List`, `Cascade:Channel:List`
  - `Task:Query`, `Task:Control`

### 前端

需要更新以下内容：

1. **路由配置**
   - 级联平台路由：`/cascade/platform` → `/device/cascade/platform`
   - 级联通道路由：`/cascade/channel` → `/device/cascade/channel`

2. **菜单引用**
   - 如果前端硬编码了菜单ID，需要更新任务管理相关ID（600→800, 601→801等）
   - 建议使用 `menu_code` 而非 `id` 进行菜单引用

3. **面包屑导航**
   - 级联功能的面包屑需要显示：设备管理 > 级联平台/级联通道

## 验证步骤

迁移完成后，执行以下SQL验证：

```sql
-- 验证级联菜单正确挂载到设备管理下
SELECT id, parent_id, menu_code, menu_name, path 
FROM tb_menu 
WHERE id IN (601, 602);
-- 预期：parent_id = 500, path 以 /device/cascade/ 开头

-- 验证任务管理使用新ID
SELECT id, parent_id, menu_code, menu_name 
FROM tb_menu 
WHERE menu_code = 'TaskManagement';
-- 预期：id = 800

-- 验证无ID冲突
SELECT id, COUNT(*) as cnt 
FROM tb_menu 
GROUP BY id 
HAVING cnt > 1;
-- 预期：无结果

-- 验证角色权限已分配
SELECT COUNT(*) FROM tb_role_menu 
WHERE role_id = 1 AND menu_id IN (601, 602, 800, 801, 80101, 80102);
-- 预期：6
```

## 回滚方案

如果需要回滚到原有结构，执行：

```sql
-- 删除新菜单
DELETE FROM tb_role_menu WHERE menu_id IN (601, 602, 60101, 60102, 60103, 60104, 60201, 60202, 60203, 800, 801, 80101, 80102);
DELETE FROM tb_menu WHERE id IN (60203, 60202, 60201, 60104, 60103, 60102, 60101, 602, 601, 800, 801, 80101, 80102);

-- 重新插入旧菜单结构（参考1.0.9之前的SQL脚本）
```

**注意**：回滚会导致前端菜单显示问题再次出现。
